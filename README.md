Цей проєкт перетворює стандартний набір SunFounder PiCar-X на розумного робота з веб-інтерфейсом та комп'ютерним зором. Він використовує нейромережу YOLO для розпізнавання дорожніх знаків у реальному часі прямо на Raspberry Pi.

> **⚙️ Апаратна особливість:** Сервоприводи камери (Pan/Tilt) навмисно відключені від плати (порти P1/P2) для забезпечення стабільного, фіксованого кута огляду під час розпізнавання знаків. Кут налаштовується вручну перед заїздом.

---

## ✨ Основні можливості

* **ШІ в реальному часі:** Використання моделі `best.onnx` (YOLOv8 Nano) для детекції об'єктів.
* **Швидкий відеопотік:** Оптимізований стрімінг через `rpicam-vid` та `MJPEG` для мінімальної затримки.
* **Ігрове керування (WASD):** Підтримка одночасного натискання клавіш (наприклад, W+A) для плавних поворотів під час руху.
* **Автономна мережа:** Робот працює як незалежна Wi-Fi точка доступу.
* **Телеметрія:** Живий моніторинг рівня заряду акумуляторів (18650) у веб-інтерфейсі.

---

## 🚀 Швидкий старт (Запуск робота)

### 1. Підключення
Робот автоматично роздає власну Wi-Fi мережу. Підключіть свій комп'ютер до неї:
* **Мережа (SSID):** `PiCarX_AI`
* **Пароль:** `supersecret`

*(Для максимальної швидкості відеопотоку переконайтеся, що Ethernet-кабель відключено).*

### 2. Запуск сервера
Відкрийте термінал і підключіться до робота по SSH:
```bash
ssh pi@10.42.0.1


cd ~/picar_yolo
source venv/bin/activate
sudo killall rpicam-vid  # Звільняємо камеру на випадок зависань
python web_yolo.py


Навчання власної моделі (Custom Dataset)

Проєкт підтримує кастомні моделі YOLOv8. Пайплайн навчання виглядає так:

    Розмітка даних: Використовується локальний сервер Label Studio (формат експорту - YOLO).

    Тренування (на Apple Silicon):
    Bash

    yolo detect train data=data.yaml model=yolov8n.pt epochs=50 imgsz=320 device=mps

    Експорт для Raspberry Pi:
    Bash

    yolo export model=runs/detect/train/weights/best.pt format=onnx imgsz=320

    Замініть файл best.onnx у корені проєкту на новий.

🛠 Траблшутинг (Поширені проблеми)

1. Машинка не реагує на команди (Помилка "GPIO busy")
Це означає, що попередній процес не завершився коректно і тримає контакти моторів.
Bash

sudo pkill -9 -f python

2. Чорний екран замість відео
Перевірте фізичне підключення шлейфу камери до Raspberry Pi:
Bash

rpicam-hello

3. Зникла Wi-Fi мережа робота
Якщо ви підключали LAN-кабель, точка доступу могла деактивуватися. Увімкніть її примусово:
Bash

sudo nmcli device wifi hotspot ifname wlan0 ssid PiCarX_AI password supersecret
